# 구축 태스크 목록

## 1. 목적

이 문서는 센터 출입기록 개선안을 실제로 구축하기 위한 작업을 n8n, Jenkins, 운영 준비 관점에서 정리한다.

## 2. 1단계: 입력 및 데이터 준비

1. Google Form 생성
2. Google Sheets 원본 시트 확인
3. `checkins_normalized` 시트 생성
4. `checkins_invalid` 시트 생성
5. 상주 외주 인력 목록 반영
6. 케이지/위치 목록 반영

완료 기준:

- 현장에서 입력 가능한 Form과 시트 구조가 준비됨

## 3. 2단계: n8n 구축

1. Google 연결 확인
2. 신규 응답 수집 워크플로우 생성
3. 필드 매핑 구현
4. 코드값 정규화 구현
5. 유효성 검증 구현
6. `submission_id` 생성
7. 승인/요청 데이터 수집 방식 확정
8. 승인/요청 데이터 적재 워크플로우 생성
9. Jenkins 입력 데이터 적재 방식 구현
10. Slack 알림 워크플로우 생성
11. 파트너 아지트 API 또는 봇 연계 방식 확정
12. 파트너 아지트 확인요청서 자동 생성 워크플로우 생성

완료 기준:

- n8n이 입력 수집, 정규화, 알림, 후속 요청 생성까지 수행 가능

## 4. 3단계: Jenkins 구축

1. `idc-access-reconciliation` Job 생성
2. 실행 주기 설정
3. 입력 데이터 위치 확정
4. 입력 로더 구현
5. 데이터 검증 및 전처리 구현
6. 승인 완료 건 필터링 구현
7. 승인번호 기준 매칭 구현
8. 보조 검증 구현
9. 상태 판정 엔진 구현
10. `reason_code` 생성 구현
11. 결과 저장 구현
12. 요약 집계 구현

완료 기준:

- Jenkins가 정기적으로 대조하고 결과를 남길 수 있음

## 5. 4단계: 운영 연계

1. Slack 채널 준비
2. IDC 담당자 수신자 확정
3. 외주 책임자 수신자 확정
4. 파트너 아지트 확인요청서 템플릿 반영
5. 파트너 아지트 회신 결과 확인 방식 정리
6. 종료 처리 방식 정리

완료 기준:

- 예외가 발생하면 실제 사람의 행동으로 이어짐

## 6. 5단계: 운영 오픈 준비

1. 적용 대상 센터별 양식/목록 반영
2. 외주 인력 안내
3. IDC 담당자 운영 기준 공유
4. 장애 시 확인 포인트 정리
5. 적용 일정 확정

완료 기준:

- 적용 대상 센터에 운영 적용 가능

## 7. 테스트 시나리오

1. 정상
2. 승인번호 누락
3. 승인번호 무효
4. 센터 불일치
5. 체크인 누락
6. 이름 불일치
7. 파트너 아지트 확인요청서 자동 생성

## 8. 최소 오픈 조건

- Google Form 운영 가능
- n8n 정규화 가능
- Jenkins 대조 가능
- Slack 알림 가능
- 파트너 아지트 확인요청서 자동 생성 가능

## 9. 결론

이번 구축의 핵심은 아래 다섯 줄이다.

- 입력이 된다
- 정규화가 된다
- 대조가 된다
- 알림이 된다
- 후속 확인요청이 자동 생성된다
