# 센터 출입기록 개선안 개요 및 도입 계획

## 1. 목적

이 문서는 가산(LG) IDC와 판교(SK) IDC의 출입기록 전산화와 자동 대조 체계를 도입하기 위한 전체 개요 문서다.

목표는 다음과 같다.

- 종이 출입기록 누락 재발 방지
- 파트너 아지트 승인/요청과 실제 현장 체크인 연결
- 예외 탐지 시 Slack 알림과 외주 확인 요청 자동화
- 감사 대응 가능한 증적과 이력 확보

## 2. 배경

감사에서 종이 출입기록과 출입요청 대조 체계에 대한 개선 필요사항이 확인되었고, 이에 따라 본 개선안을 추진하게 되었다.

핵심 문제는 다음과 같다.

- 종이 출입대장에 의존
- 승인 기록과 실제 현장 출입 기록이 자동 연결되지 않음
- IDC 담당자는 본사 근무 중심이라 실시간 확인이 어려움
- 예외 발생 시 외주 인력에게 수동으로 확인 지시해야 함

## 3. 개선 방향

핵심은 종이 출입대장을 바로 없애는 것이 아니라, `내부 디지털 체크인 기록`과 `자동 대조`를 추가하는 것이다.

운영 흐름은 아래와 같다.

1. 외부 방문 또는 관련 현장 작업에 대해 파트너 아지트 승인/요청 존재
2. 현장 출입 전 Google Form 체크인 입력
3. sandbox n8n이 응답 수집 및 정규화
4. sandbox Jenkins가 승인/요청과 체크인 기록 대조
5. 예외 발생 시 IDC 담당자에게 Slack 알림
6. 자동 후속조치 대상이면 파트너 아지트 확인요청서 자동 생성
7. 외주 인력이 사실확인 후 회신
8. IDC 담당자가 종료

## 4. 시스템 역할

### Google Form / Google Sheets

- 현장 체크인 입력
- 원본 응답 저장

### sandbox n8n

- 응답 수집
- 정규화
- Slack 알림 연계
- 파트너 아지트 확인요청서 자동 생성

### sandbox Jenkins

- 승인/요청과 체크인 대조
- 누락/위반 판정
- 결과 저장

### Slack

- IDC 담당자 알림
- 운영 예외 인지

### 파트너 아지트

- 승인/요청 기준값
- API 기반 승인/요청 조회
- 자동 후속 확인요청서 생성
- 외주 인력 확인 결과 회신

## 5. 도입 범위

우선 도입 범위:

- 외부 방문 건
- 외주 인력 확인이 필요한 출입 예외 건
- 승인번호 또는 요청번호 기준으로 식별 가능한 건

초기 제외 가능 항목:

- 종료 보고 자동화
- 사진 첨부 의무화
- 동행자 개별 상세 등록

## 6. 성공 기준

- 승인/요청 대비 체크인 누락 탐지 가능
- 승인번호 없는 체크인 탐지 가능
- 센터 불일치 탐지 가능
- IDC 담당자가 Slack으로 즉시 인지 가능
- 외주 인력 확인 요청이 파트너 아지트에 자동 생성됨
- 모든 예외에 대해 조치 이력이 남음

## 7. 우선 구현 항목

1. 체크인 Form/Sheet 구성
2. n8n 수집 및 정규화
3. Jenkins 대조 배치
4. Slack 알림
5. 파트너 아지트 자동 확인요청서 생성

## 8. 결론

이번 도입의 핵심은 단순 전산화가 아니라 `승인/요청 -> 체크인 -> 대조 -> 알림 -> 후속 확인 -> 종료` 흐름을 하나로 만드는 것이다.

이 구조가 정착되면 감사 지적사항 재발 가능성을 크게 줄일 수 있다.
